Working with the Kronos API

Workforce Central (WFC) by Kronos is an all in one suite that includes these server apps

  • Timekeeper
  • Scheduler
  • HR
  • Payroll

All which connect to a shared collection of data. This guide will cover Timekeeper. For detailed information about the API you can download the developer's guide and the developer's reference. However this page is an attempt to minimize these guides into only the bare necessities from a scheduling app's perspective. We will connect using JavaScript and AJAX.

Requirements

Before you begin you should have an understanding of

  • XML
  • AJAX
  • APIs
  • HTTP

Overview of the server

WFC is written on Java and it communicates over an XML API. Connect to the server at http://serveraddress/wfc/logon.

Creating a response

Any response you send to the server will take this generic form. Here's an overview of what each part does.

xml
A header tag that gives information on the type of xml we are sending. Always use version 1.0 and character encoding UTF-8.
Kronos_WFC
This tag will always be a parent -- at the top most level by itself. It tells the server what app we're working with and the version. Cinemark uses 5.0.
Take note on the version number, as any newer commands will not work here.
Request
Self closing tag. Multiple request tags can be in a single Kronos_WFC tag. Its attributes depend on the method being used.
<?xml version = "1.0" encoding = "UTF-8"?>
<Kronos_WFC version = "5.0">
    <Request
    />
</Kronos_WFC>
